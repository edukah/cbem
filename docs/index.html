Why Use cBEM?

  • Provides context between nested components using _

  • Improves readability and predictability
  • Enforces consistency across large-scale projects

Syntax Summary

  • block--mod--extra ‚ùå ‚Äì avoid double modifiers
  • block__ ‚ùå ‚Äì empty element name
  • __element ‚ùå ‚Äì missing block
  • --modifier ‚ùå ‚Äì missing block and element
  • block_ ‚ùå ‚Äì empty context
  • block__element-- ‚ùå ‚Äì empty modifier
  • block__--modifier ‚ùå ‚Äì invalid element-modifier pairing
  • Block__Element ‚ùå ‚Äì uppercase not allowed
